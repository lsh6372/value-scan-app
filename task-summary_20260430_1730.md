# 按官方文档修正 API 服务

## 问题
services.js 中两个 API 路径都是错误的，是自行编造的：
1. 代币列表：错误路径，应该是 `/open/v1/vs-token/list`
2. 资金异动列表：错误路径，应该是 `/open/v1/ai/getFundsCoinList`

## 修复
1. 从官方文档获取正确的 API 路径：
   - 代币列表：https://claw.valuescan.io/zh-CN/接口详情/代币列表.html
   - 资金异动列表：https://claw.valuescan.io/zh-CN/接口详情/AI%20追踪/资金异动列表.html

2. 重写 services.js：
   - `getTokenList` → `/open/v1/vs-token/list`，参数 `search`
   - `getFundsMovementList` → `/open/v1/ai/getFundsCoinList`，无参数

3. 更新 LongInflow 页面：
   - 改用正确的 API 函数名 `getFundsMovementList`
   - 搜索改为前端过滤（API 不支持搜索参数）

4. TOOLS.md 新增规则：
   - API 必须严格按官方文档
   - 禁止自行编造 API 路径
   - 添加已对接 API 清单

## 结论
构建通过，API 路径已按官方文档修正。
