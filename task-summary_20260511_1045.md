# 移动端响应式优化

## 目标
优化项目在移动端的访问体验，并为后续新增页面建立响应式规范。

## 实现

### MainLayout（全局导航栏）
- 桌面端（≥768px）：横向水平菜单
- 移动端（<768px）：汉堡按钮 + Drawer 右侧抽屉
- Logo 字体、Header padding、Header 高度均响应式

### LongInflow 页面
- 表格列响应式：`useMemo` 包裹 `columns`，根据 `isMobile` 动态生成
- 移动端隐藏次要列（首次推送、推送价格、跌幅、看涨情绪、市值、短线异动、趋势异动）
- 移动端保留核心列（#、币种、最新推送、当前币价、涨幅、类型）
- 列宽相应缩小（币种 180→120px，币价 110→80px 等）
- 表格横向滚动：桌面 x:1400，移动 x:600
- 标题栏 flex-wrap 换行
- 搜索栏缩小（输入框 200→120px，按钮 middle→small）
- 分页简化（移动端 `simple: true`）
- Card padding 响应式（桌面 16px，移动 12px 8px）

### TOOLS.md 新增规范
- 断点常量 BREAKPOINT_MOBILE = 768
- 响应式检测标准写法
- 表格列/布局/分页响应式规范
- 后续新增页面必须默认支持 PC + 移动

## 结论
构建通过，移动端已优化。后续新增页面参考规范即可默认支持移动端。
