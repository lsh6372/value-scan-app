# Cloudflare Worker SSE 代理实现

## 问题
Vercel 部署的前端无法直接连接 SSE 端点，因 CORS 跨域限制。

## 解决方案
Cloudflare Worker 作为 SSE 中转代理。

## 架构
```
浏览器 --> Cloudflare Worker --> stream.valuescan.ai SSE
       <--SSE流--
```

## 新增文件
1. `worker/src/index.js` - Worker 代码（签名 + SSE 代理）
2. `worker/wrangler.toml` - Worker 配置
3. `worker/package.json` - 依赖
4. `DEPLOY.md` - 部署指南

## 修改文件
1. `src/api/sse-sign.js` - 简化为只返回 Worker URL
2. `src/pages/MarketAnalysis/index.jsx` - 连接 Worker URL
3. `.env.example` - 新增 VITE_SSE_WORKER_URL

## 安全设计
- AK/SK 存储在 Worker 环境变量（wrangler secret）
- 前端不接触密钥，只知道 Worker URL
- Worker 内部完成签名

## 部署步骤
1. `wrangler login`
2. `wrangler secret put VALUESCAN_API_KEY`
3. `wrangler secret put VALUESCAN_SECRET_KEY`
4. `npm run deploy`
5. 配置 .env.local 的 VITE_SSE_WORKER_URL
6. 重新部署 Vercel

## 结论
代码已改造完成，部署指南见 DEPLOY.md。
