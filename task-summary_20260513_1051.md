# Telegram 推送功能迁移

## 目标
将 Python 版 market_subscribe.py 中的 Telegram 推送逻辑迁移到大盘分析 Web 页面。

## 实现

### Worker 端 (`worker/src/index.js`)
- 新增 `POST /telegram` 端点
- 从环境变量读取 `TELEGRAM_BOT_TOKEN` 和 `TELEGRAM_CHAT_ID`
- `sendTelegramMessage()` 函数调用 Telegram Bot API
- 自动分割超长消息（每段最多 4000 字符）

### 前端 API (`src/api/telegram.js`)
- `sendTelegramMessage(text)` - 调用 Worker 代理发送
- `buildSignalText(signals)` - 构建信号摘要文本

### 页面组件 (`src/pages/MarketAnalysis/index.jsx`)
- 新增 Telegram 配置区（Tg Icon + 开关 + Bot Token + Chat ID + 测试按钮）
- 配置保存在浏览器 localStorage
- 信号变化时自动推送（仅通知变化的币种）
- 首次收到信号时也推送（prevRef 为 null 时）

### 新增依赖
- Ant Design: `Switch`, `Input`, `Tooltip`

## 部署步骤（新增）
```bash
# 在 Worker 目录执行
wrangler secret put TELEGRAM_BOT_TOKEN
wrangler secret put TELEGRAM_CHAT_ID
npm run deploy
```

## 结论
Telegram 推送功能已完整迁移到前端，Worker 负责中转。信号变化时自动推送，无需人工干预。
